多值依赖(MVD)

书上对多值依赖的定义：

def 多值依赖：

​ 设$U$是关系模式$R$的属性集，$X$和$Y$是$U$的子集，$Z=R-X-Y$，小写的$xyz$表示属性$XYZ$的值。对于$R$的关系$r$，在$r$中存在元组$(x,y_1,z_1)$和$(x,y_2,z_2)$时，也就存在元组$(x,y_2,z_1),(x,y_1,z_2)$，那么称多值依赖$X\rightarrow\rightarrow Y$在模式$R$上成立。

个人觉得这个定义……非常的丑……

关于MVD的推理规则集

M1(MVD的补规则)：若$X\rightarrow\rightarrow Y$,则$X\rightarrow\rightarrow (U-XY)$

M2(MVD的增广性)：若$X\rightarrow\rightarrow Y, V \subseteq W \subseteq U$，则$WX\rightarrow\rightarrow VY$

M3(MVD的传递性)：若$X\rightarrow\rightarrow Y, Y\rightarrow\rightarrow Z$，则$X \rightarrow\rightarrow (Z-Y)$

代数结构

代数系统

代数系统：载集，运算，n元运算，运算封闭性

单位元：

  • 左单位元，右单位元； 逆元 = 左单位元 & 右单位元
  • 单位元若存在则唯一。

逆元：

  • 左逆元，右逆元(对于一个$a \in S$)，逆元 = 左逆元 & 右逆元
  • 当运算满足结合律且a有逆元时，逆元是唯一的。

零元：

  • 左零元，右零元，零元。

分配律：

  • 左分配律：$a (b + c) = a b + a c$ (称关于+有左分配律) (右类似)

代数系统的同态：

​ 同态映射：运算相容

​ $\phi(x) : S \rightarrow T$

​ 满同态映射得到两个系统同态。

​ 同态的性质的传递$S \rightarrow T​$：交换律，单位元

代数系统的同构：

​ 当$\phi(x)$是双射时，得到同构$S \cong T$，可看成是同一个代数系统。

商系统：

​ 在代数系统$[s, *]$上新定义一个相容等价关系$t$。

​ 称$[s/t, \bigoplus]$是一个商系统

群

基本概念：

​ 半群：满足结合律的代数系统

​ 拟群：含单位元的半群

​ 群：每个元素都有逆元的拟群

​ 阿贝尔群：交换群

​ 群的阶：$|G|$，有限群，无限群

定理13.5：群G满足消去律。

定理13.6：半群G是群，当且仅当下列条件满足

​ 1.对任意$g \subseteq G$ 存在左单位元(存在$e’g=g$)

​ 2.对任意$g \subseteq G$ 存在左逆元(存在$g’g = e’$)

陪集：

​ 设H为G的子群，取G中一个固定元素g，用g与H中的每个元素进行乘法运算，将其结果组成一个集合，记为g，称其为H的左陪集，同理有右陪集。

Intel Code Challenge Elimination Round F. Cyclic Cipher

思路：
对于每个数列的某个数字，其移到第一位的时间可以用$k \times len[i] + p[i]$来表示，即满足条件的时间满足$T = p(mod\ len)$
所以问题可以大概转成一坨同余方程，判断是不是有解的问题。
一开始根据tutorial的做法是开棵线段树维护方程的解，复杂度$O(NlogNlogN)$后面tle了，原因是快速乘法太慢，之后改成__int128，CF上总是编译错误。
然后看了些奇怪的代码发现其实根根本不需要维护方程的解，只需判断是否有解有可以了。
怎么判断有解呢？发现每个方程的$len \leq 40$，也就是最后合并出来的大模数分解质因数后都$\leq 40$，其实我们可以把所有的方程拆成$len$的所有以$\leq 40$的因子为模的方程，然后对于每类模相同的方程暴力判断即可。
学到了

Codeforces Round #364 (Div. 1) D. Huffman Coding on Segment

思路
离线莫队后，这道题目的核心问题是：如何快速地模拟出huffman编码后的结果。
可以发现不同的出现频率最多只有$\sqrt{2n}$种，我们将出现频率$\leq \sqrt{n}$的和大于的分开讨论：
对于$val \leq \sqrt{2n}$的，如果这样的频率数有$X_{val}(X_{val} \leq 3)$个,那么我们在合并的过程中，一定会取出其中的$\frac{X_{val} - 1}{2} \times 2$个两两合并成$2\times val$.这样做一遍之后，小于$\sqrt{2n}$的数是$O(\sqrt{n})$级别的，而对于$>\sqrt{2n}$的数，因为在合并的过程中始终有$\sum val_i=n$所以这样的数也不会超过$\sqrt{n}$.之后暴力合并即可。
另外合并也需要利用到单调的技巧。
