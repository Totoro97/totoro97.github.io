2015 ACM/ICPC Asia Regional Shanghai Online 解题报告

发表于 2015-10-03

A Puzzled Elena
题目大意: 给定一颗树，每棵树有权值，对于每个节点，询问以该点为根节点的子树中有多少个节点与该节点互质．
因为节点的权值不会超过十万，对于一个数，分解后的质数的种类数最多只有6种，想到容斥．
容斥之前，需要解决的问题是询问一颗子树中是$x$的倍数的节点有多少个．
解法比较巧妙(个人认为)：对于一个数$x$，分别记录进入该棵子树之前和之后的$x$的倍数有多少，相减即可．

B　Antonidas
题目大意:　给定一棵树，每个点上有一个字符，询问是否能够在树上走出给定字符串．
树分治，每次解决经过该点的路径是否可行．
需要维护每个点到该点是否为给定串的前缀/后缀.可以字符串哈希也(可以后缀自动机)．
然后开个布尔表查一查就可以了．

C Typewriter
题目大意: 输出不同的字符有相应的代价，选择一个字符有固定代价，复制粘贴有代价．
给定一个串，问输出给定串的最小代价．
感觉是很神奇的一道题
后缀自动机预处理 + 单调队列DP(题目有单调性质)．

D　Count the Grid
题目大意: 给定一个矩形，然后告诉你$n$个子矩形的最大值，问可能的大矩形个数．
离散化 + 状压DP….
事实告诉我们不要一看到小数据就跳到容斥的坑里．．．．

E Code Formatting
模拟题
暂时还不会做

F There was a kingdom
题目大意: 给出$n$个点,要求选定$k$个点使得围成的多边形面积最大．
这$k$个点一定在凸包上．
可以随机起点，可以倍增，具体:http://talk.icpc-camp.org/d/78-2015-f-there-was-a-kingdom/10

G A simple graph problem
暂时还不会做
会做了．．．
每个点等价，所以只考虑某一个点对答案的贡献，最后乘以$n$即可．
一开始的想法是，计算这个点的成为割点的方案数，然而并不好做
反过来搞，计算这个点不能成为割点的方案数．
三种情况不能成为割点：
1.其余$n-1$个点根本不鸟这个点．
2.这个点自娱自乐，然后其余点中有一个点鸟这个点．
3.这个点在一个环中(不是自环)，然后除开环以外不再有点鸟这个点．
复杂度$O(n)$

A Traversal
目前还没看懂题解．
看了叉姐的解法，他的挺好懂的．

题目其实是求 n 个点的，度是 2 的生成子图的数量。
设 f(i,mask) 表示考虑了前 i 个点，其中 [i−p−2,i−1] 的度序列是 mask，很明显，mask 是一个 (p+2) 位的 3 进制数。转移就是枚举 i 往前面哪些点连边，复杂度是 $100×3^{11}×3^2$。

B Best Solver
题目大意:求 ${(5+2 \sqrt{6})}^{1+2^x}$．
其中 x <= 2^31;
首先可以使用矩阵乘法(这个竟然一开始没想到)
然后就是找循环节的问题．．这玩意是玄学．
(其实我是嘴巴出来的)

C Minimum Cut
题目大意:给你一棵树，以非树边，询问一种只割掉一条树边的割边方案，使得割掉的边数最少．
跨越树边的边都要割，相当于在树边上+1.
这个无脑一点可以熟练剖分＋线段树维护．
不过简单一点的方法是可以打标记，类似于前缀和的方法．

D Dividing This Product
题目大意:求小于等于n的所有质数的乘积
详见: http://talk.icpc-camp.org/d/48-2015-d-dividing-this-product/14

E Excited Database
题目大意：给出矩阵，初始值为零，两种操作，一是给一段连续的对角线区域加上一，二是询问某一子矩形内部元素和．
感觉是挺巧妙的题目．
对对角线建线段树．维护$\sum{A_i}$和$\sum{A_i \times i}$
利用这两个维护的信息就可以对子矩形求和了．

F Fang Fang
签到题，不多说

G Matches Puzzle Game
题目大意：
问恰好用$n$根火柴棍能拼出形如$A - B = C$的等式的方案数．
DP.
一开始想到用直接减法DP，敲完后发现细节有点多，弃掉了．
移项改成加法，可以很自然地列出状态和方程．

H　Hold Your Hand
题目大意:给出$n$个8-bit串，再给出$m$个咒语，每个咒语对应一个前缀或后缀$S_i$，使用此咒语时能将所有前缀(后缀)为$S_i$的串消灭，而且有一定的代价．
求将所有串消灭的最小代价．
神奇的网络流
每个要么被前缀咒语干掉，要么被后缀咒语干掉，联想到最小割．
对前缀咒语建trie,后缀反向建trie，同时建网络流图，流量YY一下就知道了．
对于每个串，从前缀trie的对应点向后缀trie中的对应点建一条流量为INF的边．
最大流即为答案．

I Stability
题目大意:一张图，两种操作，一是删边，二是给定a,b，询问有多少个点是a,b的路径中必须经过的．保证总是连通图．
看到修改操作真有删除就很容易想到倒着搞．
因为给定的图总是连通图，不妨一开始就是一颗树，那么删边变为加边．
加边带来的影响就是跨越的点不再是必经的点．
然后就是树上的维护问题啦，一开始的点初始值都为一，操作就是把某一条链上的权值都变为零．
树链剖分＋线段树可以搞．

J Jesus Is Here
算是签到题

K Poker
题目大意:给出不多于8个值不超过13的正整数，问有多少种不同使用给定数字的后缀表达式使得计算的值为$Q$
其实很好奇为什么这道题当场做出来的人很少
官解给的是搜索，但是我没有想明白什么样的搜法可以解决交换算两次的问题
只有8个,状压DP:$f[st][val]$，st表示数字的使用状态，val表示当前计算出来的值．
转移使用快速枚举子集的技巧．

L Largest Point
签到题

M Manors
题目大意:有m个人，每个人对花园上的任意一点都会有权值，权值由给定公式算出，权值最高者占领此地．
问每个人占领的面积．
把式子划一划，发现两个人之间会有一条分界的直线．
然后就是半平面交啦．

A
把六边形补成三角形,再减去三个角就可以了

B
直接按照题面模拟暴力可以水过,然而复杂度显然是不对的.
正解利用了相当于”最小化表示”的思想.
将A,B两个串根据的题目的要求,最小化表示.
然后只需$O(n)$判断A,B串是否相等.

C
直接容斥做会爆复杂度
然而这种容斥一般都能DP
$F_i$ 表示 以第i个黑点为终点对答案的贡献
转移: $F_i = -\sum F_j \times C(x,x + y)$
其中$x$和$y$表示点i,j之间的横纵坐标之差.

D
这种题目还是不太会做
题目的意思是给你一个严格意义上的凸包(就是说凸包上的相邻三点不存在共线的情况)
然后要你求任取端点所形成的新的凸包的内部格点的个数.
这里要用到皮克定理 : 格点多边形的面积等(于内部的点数)加上(边上的点数/2)减去1;
直接枚举每一个凸包,不太科学,考虑枚举两两端点所组成的线段对答案的贡献
有向线段被用到的概率这个很好算
然后考虑$A_iA_{i+k}$它对答案的贡献,根据皮克定理也是可以算的.
这么做的复杂度是$O(n^2)$
但是k > 60时对答案的贡献是及其微小的.于是就变成了$O(n \times 60)$

//SAM
struct node {
    node *f,*nex[26];
    int ml,size,first;
    node () { ml = size = first = 0; }
} pool[maxn],*tail, *init, *rank[maxn];

void add(int ch,int len) {
    node *p = tail, *np = &pool[++tot];
    np -> ml = len;
    for (; p && !p -> nex[ch]; p = p -> f) p -> nex[ch] = np;
    tail = np;
    if (!p) np -> f = init;
    else {
        if (p -> nex[ch] -> ml == p -> ml + 1) np -> f = p -> nex[ch];
        else {
            node *q = p -> nex[ch], *just = &pool[++tot];
            *just = *q;
            just -> ml = p -> ml + 1;
            q -> f = np -> f = just;
            for (; p && p -> nex[ch] == q; p = p -> f) p -> nex[ch] = just;
        }
    }
}

void get_suffix() {
    int i,j;
    for (i = 1; i <= top; i++) rank[i] = s[i] - 'a' + 1;
    for (i = 1; i <= top; i++) wp[rank[i]]++;
    for (i = 1; i <= 30; i++) wp[i] += wp[i - 1];
    for (i = top; i; i--) SA[wp[rank[i]]--] = i;
    int u,q;
    for (q = 1; q < top; q <<= 1) {
        memset(wp,0,sizeof(wp));
        memcpy(SA_,SA,sizeof(SA));
        for (i = 1; i <= top; i++) wp[rank[i]]++;
        for (i = 1; i <= top; i++) wp[i] += wp[i - 1];
        for (i = top; i; i--)
            if (SA_[i] > q) SA[wp[rank[SA_[i] - q]]--] = SA_[i] - q;
        for (i = top - q + 1; i <= top; i++) SA[wp[rank[i]]--] = i;
        memcpy(rank_,rank,sizeof(rank));
        rank[SA[1]] = u = 1;
        for (i = 2; i <= top; i++) {
            if (judge(rank_,SA[i],SA[i - 1],q)) rank[SA[i]] = u;
            else rank[SA[i]] = ++u;
        }
    }
}

void get_height() {
    for (i = 1; i <= top; i++) {
        if (rank[i] == 1) { H[rank[i]] = 0; continue; }
        H[rank[i]] = max(H[rank[i - 1]] - 1,0);
        while (s[i + H[rank[i]]] == s[SA[rank[i] - 1] + H[rank[i]]]) H[rank[i]]++;
    }
}

void exkmp(char *A,char *B,int *nex,int *ex) {
    La = strlen(A); Lb = strlen(B);
    nex[0] = Lb;
    for (int i = 1,j = -1,p,a; i < Lb; i++, j--) {
        if (j < 0 || i + nex[i ­ a] >= p) {
            if (j < 0) j = 0, p = i;
            while (p < Lb && j < Lb && B[p] == B[j]) j++, p++;
            nex[i] = j, a = i;
        }
        else nex[i] = nex[i - a];
    }
    for (int i = 0,j = -1,p,a; i < La; i++, j--) {
        if (j < 0 || i + nex[i - a] >= p) {
            if (j < 0) j = 0, p = i;
            while (p < La && j < Lb && A[p] == B[j]) j++, p++;
            ex[i] = j, a = i;
        }
        else ex[i] = nex[i - a];
    }
}

//最小表示法
void zxbx(char *s,char *ans) {
    int L = strlen(s);
    int i,j,cot;    
    for (i = 0,j = 1,cot = 0; cot < L && i < L && j < L; ) {
        if (s[(i + cot) % L] == s[(j + cot) % L]) cot++;
        else {
            if (s[(i + cot) % L] > s[(j + cot) % L]) i += cot + 1;
            else j += cot + 1;
            if (i == j) j = i + 1;
            cot = 0;
        }
    }
    int p = min(i,j);
    for (i = p; i < L; i++) ans[i ­ p] = s[i];
    for (i = 0; i < p; i++) ans[L ­ p + i] = s[i];
    return;
}

struct point {

int sig(double a) {
    return (a < -eps ? -1 : (a > eps));
}

//默认vector 就是 point ,就不typedef了
point operator + (point a,point b) { return point(a.x + b.x,a.y + b.y); }
point operator - (point a,point b) { return point(a.x - b.x,a.y - b.y); }
point operator * (point a,double p) { return point(a.x * p,a.y * p); }
point operator / (point a,double p) { return point(a.x / p,a.y / p); }

bool operator == (const point &a,const point &b) {
    return !sig(a.x - b.x) && !sig(a.y - b.y);

//直线的参数法表示：
struct line {
    point p,v; // 直线上任意一点x 满足 x = p + v * t;
};
// 直线求交
point cross_line(line a,line b) {
    point u = a.p - b.p;
    if (!sig(a.v / b.v)) return point(INF,INF);
    double t = (b.v / u) / (a.v / b.v);
    return a.p + a.v * t;
}
// 点到直线的距离 -> 叉积得到的有向面积除以底
double dis_dot_line(point p,line A) {
    point a = A.p,b = A.p + A.v;
    point v1 = b - a, v2 = p - a;
    return fabs((v1 / v2) / length(v1));
}
// 点到线段的距离 分三种情况考虑
double dis_dot_seg(point p,point a,point b) {
    if (a == b) return length(p - a);
    point v1 = b - a,v2 = p - a,v3 = p - b;
    if (sig(v1 * v2) < 0) return length(v2);
    else if (sig(v1 * v2) > 0) return length(v3);
    else return fabs((v1 / v2) / length(v1));
}
//点在直线上的投影点 列方程求解
point dot_projection(point a,line b) {
    return b.p + b.v * ((b.v * (a - b.p)) / (b.v * b.v));
}
//计算多边形的有向面积--虽替！
double count_s(point *p,int n) {
    double ans = 0;
    for (int i = 1; i < n; i++) ans += p[i] / p[i + 1];
    ans += p[n] / p[1];
    return fabs(ans / 2.0);
}


// Link-Cut-Tree && splay

bool home(int u) { return (fa[u] && (kid[fa[u]][0] == u || kid[fa[u]][1] == u)); } 
// 判断是不是根. 是根返回false.

void rotate(int u) {
    int f = fa[u]; int k = (kid[f][0] == u);
    fa[u] = fa[f];
    if (fa[f] && kid[fa[f]][0] == f) kid[fa[f]][0] = u;
    else if (fa[f] && kid[fa[f]][1] == f) kid[fa[f]][1] = u;
    kid[f][k ^ 1] = kid[u][k];
    if (kid[u][k]) fa[kid[u][k]] = f;
    fa[f] = u; kid[u][k] = f;
    update(f);
}

void splay(int u) {
    int f,ff;
    while (home(u)) {
        f = fa[u]; ff = fa[f];
        if (home(f) && !((kid[f][0] == u) ^ (kid[ff][0] == f))) rotate(f);
        rotate(u);
    }
    update(u);
}

void access(int u) {
    int v = 0;
    for ( ; u; u = fa[u]) {
        splay(u);
        kid[u][1] = v;
        update(v = u);
    }
}

#define Pair pair<int,int>

int build(int L,int R) {
    if (L == R) { f[L] = 0; g[L] = 0; size[L] = 1; return L; }
    int u = 0;
    for (int i = L; i <= R; i++) if (dot[i].key > dot[u].key) u = i;
    if (u > L) ls[u] = build(L,u - 1);
    if (u < R) rs[u] = build(u + 1,R);    
    update(u);
    return u;
}

int merge(int a,int b) {
    if (!a || !b) return a | b;
    if (dot[a].kay > dot[b].key) {
        rs[a] = merge(rs[a],b);
        update(a); return a;
    }
    else {
        ls[b] = merge(a,ls[b]);
        update(b); return b;
    }
}

Pair split(int a,int k) {
    if (!k) return make_pair(0,a);
    if (k == size[a]) return make_pair(a,0);

    if (k == size[ls[a]]) {
        int b = ls[a]; ls[a] = 0;
        update(a); update(b);
        return make_pair(b,a);
    } else if (k < size[ls[a]]) {
        Pair p = split(ls[a],k);
        ls[a] = p.second;
        update(a);
        p.second = a;
        return p;
    } else if (k == size[ls[a]] + 1) {
        int b = rs[a]; rs[a] = 0;
        update(a);
        return make_pair(a,b);
    } else {
        Pair p = split(rs[a],k - size[ls[a]] - 1);
        rs[a] = p.first;
        update(a);
        p.first = a;
        return p;
    }
}
